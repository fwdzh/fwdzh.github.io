写了一场ARC两个题，但感觉都简单吧。还有上场的 edu ，d 不会写，相当于也没练到。

A 本来不是很会吧，然后瞎搞，复杂度快 然后过了，时间 1000 多 ms ，Limit 是 2000 ，然后看了正解，还行。

B 忘关 cerr 了 ， T 麻了 … 以后得注意，不过这也是太依赖 cph 导致的。

这把 B 花了太久，是不是一直在发呆来着约等于，C 实际上手速也很慢， A 也手速慢 + 写得也有点唐。

等会去看看 D 。

目前这个阶段，感觉补题只能是看题解了差不多。因为写不出来的题基本上都是很难很难想出来，而且很多看了题解可能都要想很久，我可能就是会被一些莫名其妙的地方卡住不理解。

C 也有点唐了，还写个二分，有点唐了 hhhh 。

后面更新一下 D 补题记录。

五月 vp 的，那时候我只是一个绿名，看题解不理解，现在是不是也差不多来着 … 真难受啊，感觉自己要玉玉了，但是回寝室了再想了想，又突然想明白了，感觉自己更唐了，但是还是很爽的。

2023ICPC澳门区域赛 Problem J 最短路 Dijkstra/BFS submission 2024/10/17 很唐的一集，但我终于看懂了aaaaaa

2023澳门区域赛J:

给定的操作是：

  1. 移动到房间

  2. set

比较容易理解的是，一个房间只会被经过一次。

我一直不理解的是，为什么 可以被看成可以直接移动到 ，从题解和别人的代码里看到。

结果我发现我真的唐爆了，题解是真的说得很清楚，但是我没懂。

题解说的是，我们可以延迟操作 2 ，假设我们是早已经做过一次操作 1 才会进行操作 2 。

但这样说，我还是不理解，也没有理解为什么队列里加的是 …

现在没事了，

是我这个 移动了之后能去的地方，如果我们的队列里加的直接就是跳过去的状态的，那么你直接就是一定可以到 的，所以我们只需要改一下 push 进去的第一个状态就可以了，然后我们就可以自由地到达每一个 ，就直接简单地 dijkstra 就行，或者 bfs 。

等下开始补 edu 的 D 题 。

感觉这种刷题的孤独的感觉，，，还挺不错的。。。

之前提过的，陪我打今年区域赛的是两个大三学长，他们工作重心都在学其他东西上。

其实我练可能也练不出啥效果 hhhh。加油吧 。很莫名其妙的感觉，感觉比刚开始学 C 语言时还孤独捏，可能是秋天到了吧。

不过这种感觉是不是也可以挺不错的。
